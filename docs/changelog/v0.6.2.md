# v0.6.2 — Unify CLI distribution: every script gets shipped to consumer projects

## Why

The framework had two ways to expose a CLI script to consumer projects:

| Pattern | Scripts | Consumer invocation |
|---------|---------|---------------------|
| Copied via `cli/protected-files.js#FRAMEWORK_CLI_FILES` | `kb-search.js`, `lean-index.js` | `node cli/<script>.js` works after install/update. |
| Subcommand-only (spawned through `cli/index.js`) | `merge-settings.js` (pre-v0.6.2), `file-size-check.js` (v0.6.1) | Required `npx ai-development-framework <subcommand>` — every call paid an npx fetch. |

Two patterns for the same problem meant docs had to remember which script lives where, and the npx-only path was strictly slower for `/evolve` and `/setup` (which run the size check on every invocation).

## Changed

- **`cli/protected-files.js`** — added `cli/merge-settings.js` and `cli/file-size-check.js` to `FRAMEWORK_CLI_FILES`. `init`/`update` now copy them into consumer projects alongside `kb-search.js` and `lean-index.js`. No code change in the install loops — the loops already iterate the constant.
- **All consumer-facing references unified to `node cli/<script>.js`**. Affected: `_shared/file-size-guard.md`, `evolve/01-reflect-and-capture.md`, `setup/01-plugin-tables.md`, `commands/setup.md`, `rules/_global.md`, `commands/merge-configs.md`, `merge-configs/04-execute.md`, `merge-configs/05-decompose.md`, `merge-strategy.md`. The npx subcommands are still wired in `cli/index.js` and called out as fallbacks for installs predating v0.6.2.

## Migration

```bash
npx ai-development-framework update
```

After update, `cli/merge-settings.js` and `cli/file-size-check.js` will be present in your project's `cli/` directory. Existing rules / commands now resolve their lint and merge invocations locally instead of through npx.
