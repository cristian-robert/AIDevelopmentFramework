# v0.6.0 — Context budgets, /setup pipeline checks, /merge-configs decomposition

Hardens the v0.5 compression baseline: codified line-count budgets with an enforcement lint, expanded `/setup` to cover the design-artifact pipeline and reference subdirs, and rebuilt `/merge-configs` so cross-version migrations and monolithic user configs land in the slim-rule + references architecture instead of being spliced inline.

## Added

- **`cli/file-size-check.js`** — line-count lint for `CLAUDE.md`, `.claude/rules/`, `.claude/commands/`, `.claude/references/`. Exit 0/1/2 = ok/soft-warn/hard-block. `--json` mode for machine consumers.
- **`.claude/references/_shared/file-size-guard.md`** — canonical budgets per file class (rules ≤60/80, commands ≤60/80, multi-phase commands ≤80/100, CLAUDE.md ≤120/150, references graded by location).
- **`.claude/references/_shared/load-conditions.md`** — vocabulary for the **Load condition** column of command step tables (always, in-dev project, KB configured, diff-driven gates, argument-driven gates).
- **`.claude/references/merge-configs/05-decompose.md`** — algorithm that walks a monolithic user file by section, classifies each section (project meta vs. domain rule vs. skill chain vs. command vs. patterns vs. user-only), proposes a destination per the framework architecture, gets per-section approval, and re-lints budgets.

## Changed

- **`/evolve`** — Step 2.5 now runs `cli/file-size-check.js` and treats hard-cap violations as commit blockers; soft warnings surface as `size-warn=W` in the one-line output. Replaces the prior "<200 lines, target <150" prose target.
- **`/setup`** — added Step 5 (design-artifact pipeline readiness: huashu-design install, `.design-system/brand-spec.md` presence and freshness, project-local `brand-extract` skill, deliverable-routing references) and Step 6 (context guardrail). New blocker line for size-check exit 2.
- **`/setup` reference (`setup/01-plugin-tables.md`)** — enumerated per-command reference subdirs (`_shared/`, `prime/`, `start/`, `setup/`, `merge-configs/`, …) and the eight expected hooks; framework-files audit now reports `[missing]` for any absent path.
- **`/merge-configs`** — new Step 3 (version detection from `.init-meta.json#previousVersion`) and Step 5.5 (decomposition). Step 1 also picks up legacy `.claude/CLAUDE.md`. New `--no-decompose` opt-out. One-liner gains `decomposed=D` and `size-warn=W` segments.
- **`merge-strategy.md`** — Cross-version migration table makes the previousVersion contract explicit (missing/<0.5 → decompose; ≥0.5 → splice unless lint flags individual files). CLAUDE.md and Rules categories now document both the splice-default and the decompose-trigger.
- **`merge-configs/04-execute.md`** — CLAUDE.md and Rules execution paths branch on the Step 3 decision (decompose vs. splice).
- **`.claude/rules/_global.md`** — references file-size-guard and load-conditions; gains a one-line File-size guardrail section.

## Removed

- **`.claude/references/knowledge-base-templates.md`** — orphan from the pre-Karpathy-KB design; only ghost references remained in old plan documents under `docs/superpowers/plans/`.

## Migration

No code-side breaking changes. Existing installs can update with:

```bash
npx ai-development-framework update
```

Then run `/merge-configs` once — Step 3 will detect any pre-v0.5 monolithic content and offer per-section decomposition. `/setup` will start enforcing the new size budgets and reference-subdir checks; lint warnings on user-edited files trigger an extraction loop in `/evolve`.
