# v0.6.1 — Fix: CLI script invocations broken in consumer projects

## Fixed

- `/merge-configs` Step 4 told agents to run `node cli/merge-settings.js …`, which fails in consumer projects because the `cli/` directory is never copied during `npx ai-development-framework init` (only `.claude/` and `CLAUDE.md` are). Same bug affected the new `cli/file-size-check.js` lint added in v0.6.0 — every place that referenced `node cli/file-size-check.js` (`/setup` Step 6, `/evolve` Step 2.5, `_global.md`, `_shared/file-size-guard.md`, `merge-configs/05-decompose.md`) would error out the first time it ran outside this repo.

## Changed

- **`cli/index.js`** — added `file-size-check` subcommand, mirroring the spawn pattern used for `merge-settings`. Exit codes from the underlying script (0/1/2 for ok/soft-warn/hard-block) propagate so callers can branch on them.
- **All consumer-facing invocations** rewritten from `node cli/<script>.js` → `npx ai-development-framework <script>`. Affected files: `_shared/file-size-guard.md`, `evolve/01-reflect-and-capture.md`, `setup/01-plugin-tables.md`, `commands/setup.md`, `rules/_global.md`, `commands/merge-configs.md`, `merge-configs/04-execute.md` (settings deep-merge), `merge-configs/05-decompose.md`, `merge-strategy.md`.
- The `node cli/merge-settings.js` mention that remains in `merge-configs/04-execute.md` is now an explicit don't-do warning, not an instruction.

## Migration

Pure doc + CLI surface fix. Update with:

```bash
npx ai-development-framework update
```
