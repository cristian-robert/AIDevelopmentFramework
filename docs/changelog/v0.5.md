# v0.5.0 — Design-artifact pipeline + adversarial-review fixes

## New

- **Design-artifact path through PIV+E.** Distinct from production code: clickable HTML prototypes, slide decks, motion, infographics, mockups now route through a dedicated branch with their own validation phase.
- **`huashu-design` integration** ([alchaincyf/huashu-design](https://github.com/alchaincyf/huashu-design), 8.1k stars). External skill installed via `npx skills add alchaincyf/huashu-design`. Documented in install guide and `/setup` health check.
- **`/brand-extract` skill.** Project-state-aware Core Asset Protocol — classifies the project as fresh / token-driven / ad-hoc, runs huashu-design's Direction Advisor for fresh projects, refuses silent codification of slop on ad-hoc projects (requires explicit `--mode=codify-as-is` opt-in). Writes canonical `.design-system/brand-spec.md` so downstream design skills don't re-ask for assets every run.
- **Independent quality-scoring subagent** for huashu-design's 5-10-2-8 asset gate. The agent that extracts assets does NOT score them — fixes the self-grading incentive break.
- **Clarifying-script reference** at `.claude/references/design-clarifying-script.md` — canonical 3-question script + routing matrix consumed by `/start` Step 0.5 and `/plan-feature` Phase 1.5. Replaces silent keyword classifiers.
- **Brand Asset Anti-Patterns** section in `.claude/references/frontend-antislop-patterns.md` — the 5-10-2-8 quality gate, "no CSS-silhouette stand-ins for product imagery" rule. Backported from huashu-design.
- **License-acknowledgement gate** at `/execute` Step 2.5a — first design-artifact dispatch per repo prints huashu-design's license summary and writes `.design-system/.huashu-license-ack` so commercial-use obligations are visible at the point of dispatch.
- **Version-pinning for external skills** via `.claude/.versions.json` — `/setup` hashes installed `huashu-design/SKILL.md` and warns on drift from the tested commit.

## Changed

- **`/plan-feature` Phase 1.5** asks instead of keyword-matching. For in-development projects with ambiguous intent, runs the canonical clarifying script and persists answers to plan frontmatter.
- **`/start` Step 0.5** routes through the same clarifying script before scope detection. Closes the gap where `/plan-feature` was the only entry point that knew about the design path.
- **`/execute` Step 2.5** now reads plan frontmatter (`branch:`, `q1`/`q2`/`q3`, `brand_spec`) instead of re-classifying. Splits into 2.5a (license ack), 2.5b (pre-flight + dispatch + design marker write), 2.5c (hybrid handoff with explicit marker clear).
- **`/validate` Phase 2.5 (5D Visual Critique)** splits **blocking** dimensions (Visual hierarchy + Functionality, ≥7/10 to pass) from **advisory** (Philosophy / Execution craft / Innovation). Same model that produces an artifact also scores it; the advisory split prevents self-correlated taste calls from gating internal-tool ship decisions. Documented override path.
- **`spec-reviewer-marker.sh` + `spec-reviewer-enforce.sh`** add a `design:<epoch>` marker state. Design-artifact dispatches no longer rely on documentation-only carve-outs — the hook itself recognizes the state. Hybrid plans must explicitly clear the marker before the implementer→reviewer enforcement re-engages on the code phase.
- **Frontend design-skill gate** (`.claude/rules/frontend.md`) branches by deliverable type (production code / design artifact / hybrid) before the existing 3-way production-UI question.

## Fixed (adversarial review)

- **P0 — Marker enforcement was prose-only.** `/execute` Step 2.5 said "skip the marker dance" but the hook had no `design)` branch. Hybrid plans could leave the marker in a poisoned state. Fixed: added `design` write state to marker.sh, `design)` exit-0 case to enforce.sh, smoke-tested all four states (design / implementer-recent / reviewer / bogus).
- **P0 — License contagion was silent.** huashu-design's commercial restrictions were mentioned once in the install guide; nothing at dispatch time. Every framework user could ship client work through it without seeing the license. Fixed: `/execute` Step 2.5a requires `.design-system/.huashu-license-ack` on first dispatch per repo.
- **P0 — `/brand-extract` issued `git commit` directly.** Violated the framework's own mandatory `/commit` rule. Fixed: skill now stages only and tells the user to run `/commit`.
- **P1 — Keyword classifier was brittle and English-only.** Hard-coded list misrouted "build a settings panel mockup that turns into a real component". Replaced with the clarifying script + frontmatter persistence.
- **P1 — Project-state mishandling.** Single happy path treated fresh / token-driven / ad-hoc projects identically. Ad-hoc projects had their slop "frozen as canonical." Fixed: project-state classifier in `/brand-extract` Phase 0.5 with three branches; Direction Advisor for fresh; explicit warning + opt-in for ad-hoc.
- **P1 — 5D Critique was a hard ship-gate using LLM-judged numeric scores.** "Innovation 6/10" blocked PRs for fine-but-conventional internal tools. Split blocking from advisory dimensions; documented override path.
- **P1 — 5-10-2-8 quality gate was self-scored.** Same agent extracted assets, scored them, and used them. Replaced with an independent scoring subagent that doesn't know it's gating its own caller. Programmatic dedup by URL hash.
- **P1 — `npx skills add` was unpinned.** Version pin via `.claude/.versions.json#external_skills.huashu-design.skill_md_sha256`; `/setup` hashes the installed file and warns on drift.
- **P2 — Fetch budget was unbounded.** `/brand-extract` could issue ~300 fetches per project. Hard cap at 30; URL-hash cache for refs; cache hits don't charge against the budget.
- **P2 — No staleness check on `brand-spec.md`.** Spec frontmatter now records `generated:` date; `/brand-extract` and `/execute` Step 2.5b check for stale specs and brand-relevant commits since the spec was written.
