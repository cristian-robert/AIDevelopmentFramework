# /create-prd — Phase 1: Brainstorm (MANDATORY)

Invoke `superpowers:brainstorming` skill to explore the idea before writing anything.

1. If `$ARGUMENTS` provided → use as starting point.
2. If not → ask: "What are you building? Give me the elevator pitch."
3. Explore through conversation:
   - What problem does this solve?
   - Who is the target user?
   - Constraints (timeline, budget, team size, tech preferences)?
   - What does success look like?
   - What's explicitly OUT of scope?
4. Propose 2–3 architectural approaches with tradeoffs.
5. Get user's choice before proceeding.
