# /plan-feature — Phase 1: Feature Understanding

1. If `$ARGUMENTS` is an issue number → `gh issue view <number>`.
2. If a description, clarify:
   - What user-facing behavior should change?
   - Acceptance criteria?
   - S/M/L/XL scope?
   - **Design artifact or production code?** (see Phase 1.5)
3. For L/XL features → invoke `superpowers:brainstorming` skill first.
4. Write user stories: "As a [user], I want [action] so that [benefit]".
